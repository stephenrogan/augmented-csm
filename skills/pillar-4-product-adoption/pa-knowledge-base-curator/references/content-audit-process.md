# Content Audit Process

## Post-Release Audit
1. Map changed features to existing help articles
2. Flag articles referencing outdated UI or deprecated workflows
3. Prioritise by traffic (high-traffic outdated articles cause the most confusion)
4. Generate update queue for the documentation team

## Quarterly Full Audit
1. Review all articles for accuracy against current product version
2. Flag articles not updated in 6+ months
3. Check for content gaps using support ticket analysis
4. Produce gap report with estimated ticket deflection per gap
