# Feature Request Lifecycle

## Stages
1. Logged: Captured from any source. Not yet reviewed by product
2. Acknowledged: Product team has seen and accepted into backlog
3. Planned: On the roadmap with estimated timeline
4. In Development: Actively being built
5. Shipped: Live and available on the customer's tier
6. Declined: Product team decided not to build (with rationale)
7. Workaround Available: Need addressable with existing capabilities

## Matching Criteria
- Exact match: Feature planned/in-development directly addresses the request
- Partial match: Related feature addresses part of the need. Note the gap
- No match: Request is new. Include in monthly feedback report

## Status Notifications
- Planned/In Development: alert CSM with timeline
- Shipped: alert CSM with release notes and contacts who requested it
- Declined: alert CSM with rationale and suggested alternative
